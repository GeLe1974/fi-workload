Setup guide — share this dashboard
1
Fork or create a GitHub repository
Create a new public repository on GitHub (e.g. fi-workload). Upload index.html as the only file needed. No build step, no dependencies.
2
Enable GitHub Pages
Go to your repo → Settings → Pages → Source: Deploy from branch → branch: main, folder: / (root). Save. Your URL will be https://yourusername.github.io/fi-workload/
3
Create a Google Sheet for your data
Create a new Google Sheet. Name the first sheet FI_Log. Set up three columns exactly:

A: date   B: hours   C: note

Row 1 = headers. Data starts from row 2. Dates must be formatted as YYYY-MM-DD. You can paste your existing entries directly.
4
Publish the Sheet as CSV
In your Sheet: File → Share → Publish to web → choose Sheet 1 → format: Comma-separated values (.csv) → Publish. Copy the URL shown — it looks like:
https://docs.google.com/spreadsheets/d/SHEET_ID/pub?gid=0&single=true&output=csv
This URL is public but obscure — no login required to read it. The Sheet itself stays private in your Drive.
5
Connect the dashboard
In this dashboard, go to Log time → Configuration → edit. Paste the CSV URL in the "Google Sheets CSV export URL" field. Click Save config. The dashboard will now fetch live data from your Sheet on every load. Viewers just open the GitHub Pages URL — no login needed.
6
Workflow going forward
You log entries directly in the Google Sheet (fastest) or via the Log time tab (then export CSV and paste into the Sheet). Anyone with your GitHub Pages URL sees the live dashboard. Data lives entirely in your Google Drive — you can revoke the published CSV link anytime to take it offline.
