Guide de déploiement
1
Créer un dépôt GitHub
Créez un dépôt public (ex. ef-workload). Déposez index.html et l'image RSS_vs_EF.png à la racine. Aucune dépendance, aucune compilation.
2
Activer GitHub Pages
Dépôt → Settings → Pages → Source : Deploy from branch → branche main, dossier / (root). URL : https://votre-user.github.io/ef-workload/
3
Créer une Google Sheet
Trois colonnes : A: date B: hours C: note
Ligne 1 = en-têtes. Données à partir de la ligne 2. Format dates : AAAA-MM-JJ.
4
Publier en CSV
Fichier → Partager → Publier sur le web → Feuille 1 → CSV → Publier.
https://docs.google.com/spreadsheets/d/SHEET_ID/pub?gid=0&single=true&output=csv
5
Connecter le tableau de bord
L'URL est déjà codée dans le fichier index.html (ligne SHEET_URL). Si elle change, mettez-la à jour directement dans le fichier et poussez sur GitHub.
6
Workflow quotidien
Ajoutez les lignes directement dans la Google Sheet après chaque session EF. Tout visiteur avec l'URL GitHub Pages voit les données en temps réel. Révoquez le lien CSV à tout moment pour couper l'accès.
